Component Breakdown:
====================

1. TuiSession (per tab)
   ├── PTY Pair (Master + Slave)
   ├── Child Process (OpenCode, htop, vim, etc.)
   ├── Background Thread (read loop)
   └── Output Buffer (Arc<Mutex<Vec<u8>>>)

2. App State
   ├── Vector of TuiSessions
   ├── Selected Tab Index
   └── Event Handler

3. UI Layer (Ratatui)
   ├── Tab Bar Widget
   ├── Content Display (scrollable)
   └── Help Bar


Execution Flow:
===============

1. Startup
   ├── Create PTY pair for each session
   ├── Spawn child process attached to PTY slave
   ├── Start background thread to read from PTY master
   └── Initialize Ratatui UI

2. Runtime
   ├── Poll for keyboard events (100ms)
   ├── Update selected tab on Tab/Arrow keys
   ├── Read output from active session's buffer
   ├── Render UI with Ratatui
   └── Repeat

3. Shutdown
   ├── Terminate all child processes
   ├── Close all PTY file descriptors
   ├── Restore terminal state
   └── Exit cleanly


Key Benefits of This Architecture:
===================================

✓ Real Process Isolation
  - Each TUI runs independently
  - No nested terminal complexity
  - Clean process lifecycle management

✓ True PTY Support  
  - Applications think they're in a real terminal
  - Full compatibility with any TUI application
  - No special integration needed

✓ Simple Tab Interface
  - Easy to understand and use
  - Familiar keyboard shortcuts
  - Fast switching between sessions

✓ Resource Efficient
  - Only active tab rendered
  - Background threads for I/O
  - Bounded memory buffers

✓ Cross-Platform
  - Works on Linux, macOS, Windows (WSL)
  - Uses portable-pty abstraction
  - Same code across platforms
